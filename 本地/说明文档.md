# 咨询项目全流程需求管理系统

这是一个面向咨询项目负责人的本地桌面管理工具，用于统一管理项目规划、年度计划、落地版本、需求任务、资金流水和阶段成果物。

## 当前能力

- 本地 SQLite 数据库，首次启动自动创建 `data/app.db`。
- 支持项目、年度计划、落地版本、需求任务、资金流水、成果物管理。
- 支持需求待规划池，可将未定版本需求分配到当前版本。
- 支持角色视角切换，客户、研发、运营等敏感角色默认隐藏预算敏感字段。
- 支持需求状态流转、操作日志、全局搜索和 CSV 导出。
- 支持受控状态机和流转历史，非法跳转或空流转说明会被拒绝。
- 支持资金四级穿透视图：项目总预算 -> 年度预算 -> 版本预算 -> 需求预算/实际消耗。
- 支持版本冻结基线、冻结需求变更审批和跨版本需求/预算比对。
- 支持流程里程碑视图，按 6 个业务阶段查看成果物补充情况。
- 支持本地 ZIP 备份和恢复，备份包含数据库与附件。
- 预算登记、版本冻结和变更审批采用数据库事务，异常时整体回滚。
- 客户、研发、运营视角隐藏资金页、详情和敏感导出；本地角色切换仅为视角预览，不是登录鉴权。

## 开发运行

```powershell
python app.py
```

## 可运行性检查

```powershell
python -m py_compile app.py
python selftest.py
```

最终业务需求和验收边界见 `docs/最终需求与部署验收文档.md` 及仓库根目录 `最终需求与部署说明.md`。

## 打包主程序

```powershell
pyinstaller --noconfirm --onefile --windowed --name ConsultingRequirementManager app.py
```

生成文件位于 `dist/ConsultingRequirementManager.exe`。

## 打包安装器

先打包主程序，然后把主程序作为资源打进安装器：

```powershell
pyinstaller --noconfirm --clean --onefile --windowed --name ConsultingRequirementManager app.py
pyinstaller --noconfirm --clean --onefile --windowed --name ConsultingRequirementManagerSetup --add-binary "dist\ConsultingRequirementManager.exe;." installer.py
```

生成的 `dist/ConsultingRequirementManagerSetup.exe` 可双击运行，并选择安装路径。
