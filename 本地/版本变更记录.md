# 版本变更记录

## v1.4.2 - 2026-07-16

### 界面与主题

- 新增云岚蓝、松石青、靛夜紫、石墨灰四套主题，可在顶部一键切换并持久化；损坏偏好自动回退默认主题。
- 统一浅色默认侧栏、按钮、输入框、表头、选中态、提示条、资金图和状态语义色，优化 `1100x680` 最小窗口布局。
- 换肤时保留当前业务页面；版本基线和跨版本比对均在原页重绘，并保持版本管理导航高亮。
- 普通标签、多选列表和看板拖拽列表统一跟随当前主题，消除系统灰底与异色选中态。

### 业务与稳定性

- 跨版本比对覆盖描述、提出人、负责人、标签、优先级、父需求、日期、状态和备注，并按角色权限追加资金与工时差异。
- 版本不足两个时改为页面内引导并提供新建入口；演示模式内置两个版本及差异需求，可直接完成比对体验。
- 资金写入统一为两位小数，修复 `0.1 + 0.2` 边界误报和负向调整归零后的浮点残留。
- 完整 GUI 自测覆盖四主题、损坏偏好、特殊页面、权限化比对、资金精度和 13 个业务页面。

## v1.4.1 - 2026-07-15

### 业务闭环

- 补充项目当前阶段、资金申报、运营服务记录和需求原需求关联，覆盖规划到运维的结构化业务链路。
- 版本基线扩展为完整需求和已审批成果物快照；冻结版本新增成果物进入变更审批。
- 跨版本比对支持当前项目不同年度版本，并按稳定业务标识高亮新增、移除和变更。
- 增加成果物类型/挂载层级、父需求循环关系和年度/版本累计预算校验。

### 稳定性与交付

- 修复权限视角、成果物导出、资金审计前后值、状态说明保留和最小窗口布局问题。
- 安装器支持安全的非交互安装和原子升级，保留已有数据且默认不自动启动。
- 自测扩展到 13 个页面及新增业务流程，并增加独立安装器回归测试。
- 完整自测、健康检查、安装器 8 项回归、新装/升级与最终 EXE 启动截图均通过。

## v1.4.0 - 2026-07-13

### 需求完整性

- 新增稳定业务需求标识，跨版本比对不再依赖可变的需求名称。
- 新增标签字典和多选标签控件，管理员可维护启用状态。
- 新增研发任务领取、预估工时、工时明细和实际工时累计。
- 版本发布时自动生成基线并锁定核心内容。
- 新增销售、项目经理、研发、运营等角色专用导出模板，所有业务导出限定当前项目。
- 新增可拖动排序、可显示/隐藏的角色看板配置。

### 界面改造

- 首页指标改为响应式网格，新增需求推进趋势图。
- 资金穿透图展示当前版本全部需求，不再只显示预算最高的 5 条。
- 无权限操作按钮不再显示，需求列表增加领取任务和登记工时入口。
- 统一状态、资金、角色提示和表格交互样式。

## v1.3.2 - 2026-07-10

### 正式运行修复

- 数据库操作日志新增 `event_id/result`，支持与 JSON 审计逐条对账和按结果检索。
- 事件 ID 使用 UUID4 和唯一索引；旧日志升级为 `legacy` 结果并增加真实旧库迁移测试。
- 修复高日志级别导致专用错误日志漏报；补充权限拒绝和健康检查失败审计。
- 自测全程使用临时数据目录，不再污染正式数据；新目录默认不生成演示项目。
- ZIP 备份/恢复增加符号链接、解压容量和异常压缩比保护。
- 安装器打包配置移除机器绝对路径。

### 页面体验

- 更新为墨绿、蓝绿、橙和红组合的企业管理台配色，统一控件、卡片、表头、状态和资金图视觉语义。
- 修复角色控件排列和表格常驻滚动条；少量数据时自动收缩，超出时才显示内部滚动条。
- 修复切页后的失效控件回调和滚轮跨窗口竞争，列宽变化后动态刷新横向滚动条。
- 完成真实窗口截图、颜色对比度检查和 11 页面回归。

## v1.3.1 - 2026-07-10

### 正式运维

- 新增 `runtime.log`、`error.log`、`audit.log` 三类文件日志，支持按大小轮转和保留份数配置。
- 审计日志采用 JSON Lines，包含主机、进程、版本、操作者、对象、操作和结果，不记录密码或密钥。
- 增加应用启动/停止、数据库失败、关键事务、权限拒绝和 Tkinter 未捕获异常日志。
- 增加 `CRM_DATA_DIR`、`CRM_LOG_LEVEL`、`CRM_LOG_MAX_BYTES`、`CRM_LOG_BACKUP_COUNT` 正式运行配置。
- 系统设置和报表页显示日志目录，支持一键打开。
- 操作日志支持按类型/关键词过滤和审计 CSV 导出。
- 应用关闭时主动关闭 SQLite 和日志句柄。

### 测试

- 自测真实验证三类日志写入、错误分流和审计 JSON 解析。
- 新增正式运行目录 `healthcheck.py` 健康检查。
- 新增成果物附件双击/按钮打开和打开审计。
- 本地 11 个页面、业务事务和日志测试全部通过。

## v1.3.0 - 2026-07-10

### 新增

- 新增受控需求状态机、状态历史和必填流转说明。
- 新增版本基线与需求核心快照，冻结需求修改/删除进入变更审批。
- 新增预算正常、超分配和实际超支提示。
- 新增最终需求、部署和验收文档。

### 优化

- 预算登记、版本冻结、变更审批改为事务操作，异常整体回滚。
- 需求状态、状态历史和操作日志改为单一事务。
- 冻结版本禁止继续分配或调整预算，只保留实际消耗登记。
- 客户、研发、运营视角进一步隐藏资金页面、详情、搜索和导出中的敏感字段。
- 长表单支持滚动和屏幕高度约束，表格金额/文本排序更稳定。
- 成果物挂载校验、业务阶段统计、跨版本比对和编号唯一性优化。
- 年度、有效日期格式和版本起止顺序增加业务校验。

### 安全与修复

- ZIP 恢复增加路径、数据库完整性和结构校验，并自动创建恢复前快照。
- 修复临时目录与应用数据跨盘时无法替换、恢复中断可能留下半完成数据的问题。
- CSV 导出增加公式注入转义。
- 修复冻结变更审批只更新审批状态但未应用业务内容的问题。
- 修复需求资金可从普通编辑入口绕过资金流水的问题。

### 测试

- 隔离临时数据库覆盖资金事务、非法流水回滚、冻结基线和审批应用。
- 11 个页面冒烟测试全部通过。

## v1.1.0 - 2026-06-08

### 新增

- 新增 `ConsultingRequirementManagerSetup.exe` 安装器。
- 双击安装器后可选择安装路径。

- 源码新增 `installer.py` 和 `ConsultingRequirementManagerSetup.spec`。

### 调整

- `exe` 目录调整为只保留安装器，便于用户直接双击安装。

- README 增加安装器打包说明。

### 说明

- 主程序仍为 `ConsultingRequirementManager.exe`，由安装器复制到用户选择的安装目录。
- 运行数据默认保存在安装目录下的 `data` 文件夹。
