# 本地 SQLite 版最终需求与部署验收文档

基线日期：2026-07-10

完整业务需求、状态机、资金规则和双版本边界见仓库根目录 `最终需求与部署说明.md`。本文只说明本地版的运行、数据和验收要求。

## 1. 定位

本地版适用于单人离线管理、方案演示和轻量项目。数据、附件、导出和备份均保存在应用目录的 `data/` 下，不依赖网络数据库。

本地版右上角角色选择器用于预览不同角色的工作台和字段视图，不进行登录认证。任何可以访问本机数据目录的用户都可能读取 SQLite 数据库，因此不能把该角色切换当作正式权限隔离。

## 2. 已实现主链路

- 规划项目、年度计划、落地版本、需求任务的上下文联动。
- 待规划需求池、版本分配、冻结限制和跨版本比对。
- 受控状态机、必填流转说明和状态历史。
- 项目、年度、版本、需求四级资金展示和超支预警。
- 资金流水原子登记，需求资金不能通过普通编辑直接覆盖。
- 版本冻结时原子生成基线和需求核心快照。
- 冻结需求编辑/删除进入变更申请，审批与业务修改原子提交。
- 项目六阶段里程碑、成果物挂载和验收材料提醒。
- 角色工作台、敏感资金字段隐藏、搜索、排序、下钻和 CSV 导出。
- SQLite 与附件 ZIP 备份、安全路径检查、数据库完整性检查和恢复前快照。

## 3. 目录

```text
本地/
  app.py
  selftest.py
  data/
    app.db
    attachments/
    backups/
    exports/
```

`data/` 是运行数据，不应覆盖式发布，也不应提交到 Git。

## 4. 运行与自测

```powershell
cd 本地
python app.py
```

```powershell
python -m py_compile app.py selftest.py installer.py
python selftest.py
```

`selftest.py` 包含隔离临时数据库场景，会验证资金事务、非法流水回滚、版本冻结基线、变更审批、非法非有限金额、权限规则和全部页面打开。

## 5. 备份与恢复

管理员视角在“报表导出”中创建 ZIP 备份。备份包含 `app.db` 和附件。

恢复前系统会：

1. 拒绝包含绝对路径或 `..` 的不安全 ZIP。
2. 检查 `app.db` 是否存在。
3. 执行 SQLite `PRAGMA integrity_check` 和关键表检查。
4. 自动创建恢复前安全快照。
5. 替换数据库和附件并提示重新启动。

恢复仍属于高风险运维操作，应先复制整个 `data/` 到独立磁盘，并在副本中验证备份可打开。

## 6. 验收清单

- [ ] 新建项目、年度、版本时预算层级校验正确。
- [ ] 新需求可进入待规划池并分配到未冻结版本。
- [ ] 非法状态跳转和空流转说明被拒绝。
- [ ] 分配预算超过版本预算被拒绝，实际超支出现二次确认。
- [ ] 冻结后生成基线，编辑/删除自动进入变更审批。
- [ ] 客户、研发、运营视角不显示资金详情或资金页。
- [ ] 成果物挂载对象错误时不复制附件。
- [ ] CSV 以 Excel 打开时公式起始文本不执行为公式。
- [ ] ZIP 备份和恢复已使用副本完成演练。
- [ ] `python selftest.py` 全部通过。

## 7. 不适用场景

本地版不适用于多人并发、跨电脑共享、正式账号审计、强行级权限或公网访问。这些场景使用 MySQL 远程版，并在目标 Linux/网络/附件环境完成正式验收。
