# 需求实现核对与差异分析

> 本文保留 2026-07-05 的阶段性核对结果。2026-07-10 已补齐受控状态机、版本核心基线快照、冻结变更实际应用、事务化预算和安全恢复；最新结论以 `最终需求与部署验收文档.md` 和仓库根目录 `最终需求与部署说明.md` 为准。

核对日期：2026-07-05

代码范围：`app.py`、`selftest.py`、`说明文档.md`、`版本变更记录.md`

## 1. 总体结论

当前代码已具备“咨询项目全流程需求管理系统”的本地桌面 MVP 能力，主链路已经贯通：

项目规划 -> 年度计划 -> 落地版本 -> 需求任务 -> 资金流水 -> 阶段成果物 -> 搜索/导出/备份/审计。

系统目前更适合作为单机版项目负责人工作台使用，已能支撑需求收口、版本隔离、资金四级穿透和成果物追踪。严格的多用户登录、菜单级权限和管理员拖拽配置看板仍属于后续增强项。

## 3. 当前优先级建议

### P0 已满足

- 项目、年度、版本、需求、资金、成果物主数据链路。
- 需求池与待规划池。
- 版本隔离视图。
- 资金四级穿透基础可视化。
- 角色视图和敏感字段隐藏。
- 操作日志和自测脚本。

### P1 建议继续增强

- 标签字典和多选标签控件。
- 状态流转规则，例如只能按“草稿 -> 规划中 -> 已排期 -> 研发中 -> 待验收 -> 已上线运维”推进。
- 冻结版本的完整基线快照。
- 按角色导出的客户版、销售版、研发版报告模板。
- 成果物挂载对象选择器，减少手工输入对象 ID。

### P2 后续产品化

- 多用户登录和密码策略。
- 菜单级、字段级、数据行级 RBAC。
- 管理员可配置看板组件。
- 趋势图、周期统计和更完整的预算执行分析。

## 4. 本次核对后的代码整理

- 首页新增角色专属工作台区域，不同角色看到不同指标和清单。
- 表格金额字段统一格式化为千分位金额。
- 版本冻结字段由 `0/1` 展示为“否/是”。
- 指标卡支持左侧强调色，提高预算、风险、完成状态的视觉辨识度。
- 右侧主内容区改为可滚动布局，避免长页面内容被窗口底部截断。
- 首页角色工作台、需求状态分布和资金指标改为网格化卡片布局，减少横向挤压。
