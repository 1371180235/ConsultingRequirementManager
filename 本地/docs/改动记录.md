# 改动记录

## 2026-07-10

### 日志与正式运维增强

- 新增 `data/logs/runtime.log`、`error.log`、`audit.log`，分别记录运行、异常堆栈和 JSON Lines 审计事件。
- 三类日志按大小轮转，默认 10MB、保留 30 份，支持环境变量调整级别、大小和份数。
- 增加启动、停止、SQLite 失败、关键事务、权限拒绝和未捕获界面异常记录；日志不记录 SQL 参数、密码或密钥。
- 增加 `CRM_DATA_DIR`，支持程序安装在只读目录时把运行数据放到受控可写目录。
- 本地附件记录改为相对数据目录，确保 `CRM_DATA_DIR` 位于程序目录外时仍可上传和迁移。
- 成果物列表支持按钮或双击打开附件，打开操作写入数据库和文件审计。
- 系统设置和报表页展示日志目录，应用退出时主动关闭数据库及日志句柄。
- 数据库操作日志新增对象类型、关键词过滤和受权限控制的审计 CSV 导出。
- 安装器同步创建 `data/logs`。
- 自测新增运行/错误/审计日志真实写入、错误分流和审计 JSON 解析。
- 新增 `healthcheck.py`，检查真实 SQLite 完整性、关键表、可写目录、日志和磁盘空间。
- 新增仓库根目录 `日志与运维说明.md`。
- 修复 `CRM_LOG_LEVEL=CRITICAL` 会连带过滤 `error.log` 中普通错误的问题。
- 数据库操作日志新增 `event_id/result`，与 JSON 文件审计使用同一事件 ID；健康检查失败和权限拒绝记录失败结果。
- 审计事件 ID 改为 UUID4 并建立非空唯一索引；旧版操作日志结果迁移为 `legacy`，自测真实构造旧八列表验证升级分支。
- 操作日志页面显示结果与事件 ID，关键词可检索事件 ID和结果。
- `selftest.py` 改为全程使用临时 `CRM_DATA_DIR`，不再写正式数据库和日志。
- 新数据目录默认不再写演示项目，仅 `CRM_SEED_DEMO_DATA=1` 时生成演示数据。
- ZIP 备份跳过符号链接；恢复增加符号链接、解压总量、临时盘空间和异常压缩比检查。

### 本地正式界面配色与滚动细节

- 将深蓝灰通用后台配色调整为墨绿导航、低饱和蓝绿主操作、白色内容面和分离的绿/橙/红状态色，降低长时间使用的视觉负担。
- 统一导航悬停/选中、按钮、输入框、下拉框、卡片边框、表头、选中行、提示条和资金拓扑图颜色。
- 修正顶部角色标签与下拉框顺序，缩小搜索按钮，优化品牌区、页头分隔线和底部状态栏层级。
- 关键前景/背景组合对比度实测达到 WCAG AA 正文要求；警示、成功和失败不只依赖颜色表达。
- 表格按实际数据量收缩；纵向和横向滚动条仅在内容超出时显示，解决单行数据仍出现三重滚动条和大面积空白的问题。
- 修复页面切换后延迟滚动条回调访问已销毁控件的 `bad window path name`，并增加“立即切页后执行 idle”的回归场景。
- 鼠标滚轮按指针所在表格或外层页面路由，弹窗不再同时带动被遮挡的主页面；拖动列宽后重新判断横向滚动条。
- 使用真实 Tk 窗口截图检查 1296x799 和最小 1100x680 视口，并重新执行 11 页面冒烟测试。

### 业务逻辑

- 增加受控需求状态机和状态历史，流转必须填写原因；状态、历史和日志在同一事务提交，并用原状态条件更新避免覆盖。
- 增加版本基线表、需求核心快照表和冻结变更载荷表。
- 版本冻结改为事务操作：基线、需求快照、冻结状态和日志整体提交或回滚。
- 冻结需求修改/删除进入变更申请；审批通过后实际应用修改或软删除，审批和业务更新整体提交。
- 资金流水改为事务登记；分配预算、实际消耗和调整金额必须关联需求。
- 冻结版本只允许继续登记实际消耗，禁止绕过基线继续修改预算分配或调整金额。
- 增加项目/年度/版本预算层级校验、有限数值校验、版本超分配拦截和实际超支二次确认。
- 年度限制在 2000-2100，计划和需求日期统一校验 `YYYY-MM-DD` 有效日期，版本结束日期不得早于开始日期。
- 普通需求编辑不再直接修改已分配预算和实际消耗。
- 修复里程碑成果物按项目、年度、版本和需求层级统计错误。
- 成果物上传前验证挂载对象，数据库写入失败时清理已复制文件。
- 本地附件路径改为相对应用目录保存，附件编号和资金流水编号增加微秒避免碰撞。
- 跨版本需求比对改为按标准化需求名称匹配。

### 安全与恢复

- CSV 导出增加公式注入转义。
- ZIP 恢复增加路径穿越检查、`app.db` 必需检查、SQLite 完整性检查和关键表检查。
- 恢复前自动生成数据库与附件安全快照。
- 恢复文件先暂存到目标数据盘，再通过目录/文件改名替换；附件或数据库失败时自动回滚，修复跨盘 `os.replace` 和半恢复风险。
- 备份/恢复入口限制为管理员视角，变更审批和系统日志按角色限制。

### 页面与权限

- 长表单增加滚动区域和屏幕高度约束，避免按钮超出屏幕。
- 表格排序支持金额与文本混合值，冻结版本、预算正常/超支和审批冲突增加提示条。
- 客户、研发人员、运营人员视角隐藏首页、项目、年度、版本、需求详情、搜索和比对中的资金信息，并阻止进入资金页或敏感导出。
- 本地版角色仍为工作视角预览，文档明确其不构成登录鉴权。
