# 咨询项目全流程需求管理系统业务说明

更新日期：2026-07-16

本文是项目唯一的业务说明文档，描述本地版与远程版共同的业务能力、实际使用规则和交付边界。Linux 服务器的安装命令、环境要求和访问地址配置见 `远程/部署说明.md`。

## 1. 产品定位

系统面向咨询项目负责人，以资金规划为主轴、落地版本为隔离边界、需求任务为最小管理颗粒度，统一管理从宏观规划到上线运营的全过程。

统一业务链路：

`规划项目 -> 年度计划 -> 落地版本 -> 需求任务 -> 资金/成果物 -> 验收 -> 运维运营`

本地版和远程版使用同一套业务规则、页面能力和数据口径。两者只在运行方式、数据存储位置、身份认证和多人协作能力上不同：

- 本地版是 Windows 单人单机程序，数据保存在用户选择的安装路径。
- 远程版完整运行在 Linux 服务器，用户仅使用浏览器访问 `https://业务域名`。

## 2. 业务角色

| 角色 | 核心视图 | 主要业务操作 |
| --- | --- | --- |
| 管理员 | 全部业务、用户权限、审计 | 维护基础数据；远程版预建账号、分配角色、客户项目授权和密码重置 |
| 咨询负责人 | 项目规划、版本、需求、资金、审批 | 创建项目/年度/版本，统筹需求和资金，审批版本变更 |
| 客户 | 已授权项目、本人需求、版本进展 | 提交新需求、查看处理状态和版本规划；不查看资金成本和研发工时 |
| 销售 | 项目进展、资金申报、交付材料 | 提交需求、维护资金申报、上传成果物；不查看研发内部工时 |
| 项目经理 | 版本交付、验收、投入、风险 | 管理需求状态、版本投入、成果物和验收 |
| 研发人员 | 版本任务、优先级、工时 | 领取任务、更新研发状态、填报本人任务工时；不查看资金成本 |
| 运营人员 | 上线问题、推广、维护记录 | 登记线上问题和运营服务，关联原需求与上线版本；不查看资金成本 |

本地版不登录，右上角角色选择器用于单人预览不同角色的工作台和字段视角，不构成身份隔离。远程版由登录账号确定角色，权限由服务端强制校验，不能通过浏览器修改请求绕过。

## 3. 六阶段项目流程

| 阶段 | 核心工作 | 主要成果物 | 固定挂载层级 |
| --- | --- | --- | --- |
| 1. 宏观规划 | 明确项目方向和总体资金盘 | 可研报告 | 项目 |
| 2. 规划细化 | 拆分年度任务、痛点、功能和预算 | 分年任务申报书 | 年度计划 |
| 3. 建设落地 | 细化功能点和交付要求 | 任务书方案、需求任务表、任务清单 | 落地版本 |
| 4. 招投标 | 明确标的范围和招应标要求 | 招标文件、应标文件 | 落地版本 |
| 5. 项目交付验收 | 核对版本交付并归档总结 | 验收报告、项目总结 | 落地版本 |
| 6. 运维运营 | 处理上线问题、推广和维护 | 运维反馈、运营反馈 | 需求任务 |

流程里程碑页按六个阶段显示当前进度、成果物数量和验收缺口。当版本需求已经进入待验收、上线或关闭状态，但缺少已审批的验收报告/项目总结时，系统给出补充材料提醒。

## 4. 统一功能范围

### 4.1 多角色工作台

- 顶部统一选择当前项目、年度和版本，切换后页面数据联动刷新。
- 首页显示角色待办、项目指标、需求状态分布、推进趋势和最近需求。
- 支持专业蓝、清雅绿、暖灰橙、深色模式四套主题，一键切换并保存偏好。
- 无权限的页面、按钮和敏感字段不显示；远程版同时由 API 再次校验权限。

### 4.2 项目、年度与版本

- 项目保存项目编号、名称、总预算、当前阶段和说明，项目编号唯一。
- 年度计划归属一个项目，记录年度预算、业务痛点和年度目标。
- 落地版本归属一个年度计划，记录版本编号、目标、预算和状态。
- 项目内年度预算合计不能超过项目总预算；年度内版本预算合计不能超过年度预算。
- 顶部项目切换联动年度，年度切换联动版本，避免跨项目、跨年度误挂数据。

### 4.3 版本隔离、基线与比对

- 每条需求、版本成果物、版本预算和基线均明确关联落地版本。
- 版本冻结时生成包含版本、需求、预算和成果物的完整基线，冻结后不能直接修改。
- 冻结版本的调整必须提交变更申请，经有权角色审批后执行，并生成新基线和审计记录。
- 跨版本比对至少需要同一项目存在两个版本；版本不足时显示原因和新建第二版本入口。
- 比对可跨年度选择版本，按稳定需求标识展示新增、移除、修改和一致项，并展示有权限用户可见的预算、成本和工时差异。

### 4.4 需求全生命周期

- 所有角色提交的需求进入统一需求池，创建时必须关联规划项目。
- 版本未确定时进入待规划池，可暂不关联年度和版本；分配到未冻结的落地版本时，系统按目标版本补齐所属年度。
- 完成规划后的需求关联所属规划、落地版本、来源/对接角色和占用资金四个维度；待规划需求会明确显示尚未分配版本和资金。
- 标签来自统一标签字典并支持多选；支持业务痛点、功能优化、运维 Bug、招投标要求等分类。
- 可关联同项目的原需求；系统拒绝关联自身、跨项目关联和循环关联。
- 研发可领取已排期任务并填报预估/实际工时。

需求状态按受控状态机流转：草稿、规划中、已排期、研发中、待验收、已上线运维、已关闭、已驳回、已挂起、已取消、变更中、退回修改。每次状态变更必须填写说明，并与状态历史、操作日志在同一事务保存。

### 4.5 资金全链路

- 按“项目总预算 -> 年度预算 -> 版本预算 -> 需求资金”四级展示资金流向。
- 支持计划预算、需求分配、实际消耗和调整流水。除调整金额可正可负且不能为 0 外，其余流水金额必须大于 0；所有金额必须有限、层级一致，调整后需求分配预算不能小于 0。
- 资金树可逐级展开和下钻，高亮预算超限、执行偏差和未分配资金。
- 客户和研发不查看资金成本；销售不查看研发工时；管理员、咨询负责人和项目经理可按职责查看完整投入。
- 资金申报独立管理草稿、提交、审批中、批复、驳回和拨付状态。

### 4.6 成果物、运营、搜索与报表

- 成果物按六阶段类型上传，并校验项目、年度、版本、需求的挂载层级。
- 运营服务记录支持线上 Bug、问题解答、推广活动、维护记录和功能建议，可关联原需求及版本。
- 全局搜索可按需求编号、名称、描述、标签、项目和版本定位数据。
- 支持需求、资金、资金申报和成果物等 CSV 导出，导出内容按角色隐藏敏感字段，并防止表格公式注入。
- 关键业务新增、修改、删除、状态流转、预算调整、审批和权限拒绝均记录审计信息。

## 5. 本地版交付与使用

本地版最终交付文件为：

`本地/ConsultingRequirementManagerSetup.exe`

客户只需使用 Windows 10/11 64 位电脑，不需要安装 Python、SQLite、MySQL 或其他数据库。使用步骤如下：

1. 双击 `ConsultingRequirementManagerSetup.exe`。
2. 阅读《软件最终用户许可协议》，勾选“我已阅读并接受”后才能进入下一步；拒绝协议不会创建安装目录。
3. 自主选择一个可读写的安装路径，例如 `D:\ConsultingRequirementManager`，然后完成安装。
4. 安装后从安装目录启动主程序；程序首次运行自动创建本地数据库。

默认目录结构：

```text
用户选择的安装目录/
  ConsultingRequirementManager.exe
  UninstallConsultingRequirementManager.exe
  EULA_zh-CN.txt
  data/
    app.db
    attachments/
    backups/
    exports/
    logs/
```

数据库、附件、导出、备份和日志全部位于所选安装路径的 `data/`。覆盖安装到同一路径时保留已有 `data`。卸载时由用户选择“保留业务数据”或“同时删除业务数据”；默认保留，删除数据必须再次确认。重要数据仍应定期导出 ZIP 备份并验证可恢复。

本地版用于单人单机，不支持多人同时使用、跨电脑共享数据或正式账号权限隔离。

## 6. 远程版交付与使用

远程版的 Web 前端、服务端 API、MySQL 数据库和附件目录全部部署在 Linux 服务器。用户设备只需要主流浏览器，通过以下形式访问：

`https://业务域名`

浏览器只与 HTTPS Web/API 通信，不安装客户端，不持有 MySQL 地址、端口、账号或密码。MySQL 8.4 LTS 仅在服务器容器内部网络提供服务，不直接暴露到公网。

远程账号规则：

- 不开放自助注册，由管理员在“用户与权限”页面预建账号并分配一个固定角色。
- 客户账号由管理员分配可访问项目白名单；内部角色按单组织模式使用。
- 普通用户忘记密码由管理员重置；管理员忘记密码由另一名管理员重置。
- 新建或重置密码后，用户首次登录必须修改临时密码。
- 连续登录失败达到限制后暂时锁定账号，降低暴力尝试风险。
- 同一账号同一时间只保留一个有效会话；后登录成功后，旧浏览器会话在下一次请求时立即失效并回到登录页。
- 停用账号、修改角色或重置密码会使原会话失效。

正式环境建议至少保留两名启用的管理员。服务器环境、域名解析、HTTPS、备份和升级步骤以 `远程/部署说明.md` 为准。

## 7. 两版差异边界

| 维度 | Windows 本地版 | Linux 远程版 |
| --- | --- | --- |
| 业务功能 | 与远程版一致 | 与本地版一致，另提供真实用户与权限管理 |
| 使用方式 | 双击 Setup 安装后运行桌面程序 | 浏览器访问 `https://业务域名` |
| 使用人数 | 单人单机 | 同一组织多人协作 |
| 数据库 | 程序内置 SQLite，随安装路径保存 | 服务器内 Oracle MySQL 8.4 LTS |
| 数据库凭据 | 用户无需配置数据库 | 只保存在服务器，浏览器不可见 |
| 身份权限 | 不登录，角色仅用于视图预览 | 服务端登录、RBAC、客户项目白名单 |
| 附件 | 所选安装路径的 `data/attachments` | Linux 服务器持久化目录 |
| 备份 | 应用内 ZIP 备份数据库和附件 | 服务器分别备份 MySQL、附件和部署配置 |
| 网络 | 可完全离线 | 必须能访问部署域名 |

## 8. 演示验收主线

演示或验收时按以下主线操作，可覆盖核心需求：

1. 新建项目并设置总预算。
2. 新建年度计划并拆分年度预算。
3. 在同一项目下创建至少两个落地版本。
4. 新建待规划需求，选择多个标签，关联来源角色和原需求，再分配到版本。
5. 领取研发任务并按状态机推进到待验收/上线。
6. 登记资金分配和实际消耗，在资金树中逐级穿透到需求。
7. 冻结版本生成基线，提交并审批一条版本变更。
8. 对比两个版本，查看新增、移除、修改及预算/工时差异。
9. 上传六阶段成果物，登记运营问题并关联上线需求。
10. 使用全局搜索、CSV 导出和审计日志核对完整链路。

本地版额外验收协议门禁、可选安装路径、覆盖升级保留数据和卸载数据选择；远程版额外验收账号预建、首次改密、角色权限、客户项目授权、单账号单会话和 HTTPS 域名访问。
