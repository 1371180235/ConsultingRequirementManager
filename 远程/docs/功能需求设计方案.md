# 咨询项目全流程需求管理系统功能需求设计方案

## 1. 业务背景

系统面向咨询项目负责人，用于统筹咨询项目从宏观规划、年度拆分、建设落地、招投标、交付验收到上线运维的全流程。核心业务目标是围绕客户整体资金规划，将宏观规划拆解到多个年度计划、多个落地版本和具体需求任务，并能持续跟踪需求状态、版本归属、成果物和资金占用。

## 2. 使用角色

| 角色 | 核心关注点 | 典型操作 |
| --- | --- | --- |
| 咨询负责人 | 项目全局统筹、需求收口、资金与版本总览 | 创建项目、规划年度任务、管理版本、同步各角色进展 |
| 客户 | 需求响应、项目改进计划、版本规划 | 提交需求、查看反馈、查看项目进度 |
| 内部销售 | 资金申报、项目进展、投入汇总 | 查看进度、导出报告、同步资金情况 |
| 项目经理 | 版本交付、验收、项目投入 | 维护版本交付状态、确认需求完成度、补充验收材料 |
| 研发交付团队 | 任务优先级、研发状态、交付范围 | 领取任务、更新状态、评估工期和成本 |
| 运营服务团队 | 上线运维、问题反馈、推广维护 | 提交线上问题、关联原需求和版本、维护运营反馈 |
| 管理员 | 基础数据、权限、看板和系统配置 | 管理角色权限、配置看板、备份恢复、审计日志 |

## 3. 业务流程与成果物

| 阶段 | 核心工作 | 输出成果物 | 建议挂载对象 |
| --- | --- | --- | --- |
| 1. 宏观规划 | 对项目做整体方向论述 | 可研报告 | 规划项目 |
| 2. 规划细化 | 可研批复后拆分年度任务，明确业务痛点、功能点和预算 | 分年任务申报书 | 年度计划 |
| 3. 建设落地 | 细化功能点，明确执行要求，整理交付任务 | 任务书方案、需求任务表、任务清单 | 落地版本 |
| 4. 招投标 | 明确标的工作范围与要求，完成招标/应标准备 | 招标文件、应标文件 | 落地版本 |
| 5. 项目交付验收 | 配合项目经理完成项目验收 | 验收报告、项目总结 | 落地版本 |
| 6. 运维运营 | 支持上线后的问题处理与运营推广 | 运维反馈、运营反馈 | 需求任务或落地版本 |

## 4. 设计目标

系统核心逻辑为：以资金规划为主轴，以版本隔离为边界，以需求实例为颗粒度，通过角色看板实现协同。系统需实现从宏观预算到微观需求的全链路闭环追踪。

关键目标：

- 统一收口多来源需求，减少微信、邮件、文档等零散信息遗漏。
- 建立项目、年度、版本、需求、资金、成果物之间的结构化关联。
- 支持按角色展示差异化看板，减少重复手工汇报。
- 以版本为隔离边界，避免需求、预算、成果物在不同版本间混淆。
- 提供资金四级穿透视图，展示总预算到单个需求的资金流向。
- 通过日志、权限和基线机制保证过程可追溯。

## 5. 核心数据模型

| 实体 | 定义 | 关键字段 |
| --- | --- | --- |
| 规划项目 | 顶层项目实体，承载客户、背景、总预算和可研成果 | 项目编号、项目名称、客户名称、项目背景、总预算、状态 |
| 年度计划 | 归属于规划项目，用于年度任务和资金拆分 | 年度、计划名称、年度预算、业务痛点、计划说明、状态 |
| 落地版本 | 归属于年度计划，是需求和成果物隔离的核心边界 | 版本编号、版本名称、版本目标、版本范围、版本预算、冻结状态、计划/实际周期 |
| 需求任务 | 最细颗粒度业务对象，统一承接所有来源需求 | 需求编号、名称、描述、来源角色、提出人、对接人、类型、标签、优先级、状态、预算、所属项目/年度/版本 |
| 资金流水 | 记录预算分配、实际消耗和调整过程 | 流水编号、项目、年度、版本、需求、类型、金额、说明、操作人、发生时间 |
| 成果物 | 记录各阶段产物和附件 | 成果物编号、名称、类型、文件路径、挂载对象、版本号、说明、上传人 |
| 用户/角色 | 用于登录、权限和看板过滤 | 用户名、展示名、角色、启用状态 |
| 操作日志 | 记录关键数据变更 | 操作人、操作时间、对象、操作类型、前值、后值、说明 |
| 变更申请 | 版本冻结后的修改入口 | 版本、需求、标题、原因、影响范围、审批状态、申请人、审批人 |

## 6. 功能模块设计

### 6.1 多角色工作台与看板管理

不同角色进入系统后默认看到与其职责相关的项目、版本、需求和资金信息。

| 角色视角 | 默认看板组件 | 可执行操作 |
| --- | --- | --- |
| 客户视角 | 项目整体进度、需求处理状态、最新改进计划 | 提交新需求、查看需求反馈、查看版本规划 |
| 内部销售视角 | 资金申报进度、项目整体进展、团队投入汇总 | 查看预算执行率、导出项目进展报告 |
| 项目经理视角 | 版本交付进度、需求研发状态、项目总投入 | 确认需求完成度、维护版本投入 |
| 研发交付视角 | 待办任务清单、版本需求池、优先级队列 | 领取任务、更新状态、评估工期 |
| 运营服务视角 | 上线版本列表、线上问题反馈池、推广维护记录 | 提交线上问题、关联原需求版本 |
| 管理员视角 | 全局指标、系统配置、操作审计 | 管理角色、配置看板、备份恢复 |

功能要求：

- 支持管理员配置角色可见菜单、字段和看板组件。
- 支持按角色过滤需求、成果物和资金字段。
- 支持全局搜索需求编号、名称、描述、标签、项目、版本。
- 搜索结果需展示需求所在项目、年度、版本、当前状态、预算占用和负责人。

### 6.2 需求全生命周期管理

功能要求：

- 所有角色提交的需求统一进入需求池。
- 需求创建时需关联项目，可选择年度计划和落地版本。
- 未确认版本的需求进入“待规划池”，后续可批量分配到版本。
- 需求支持多标签分类，标签可自定义。
- 需求支持优先级管理和负责人管理。
- 需求状态需与业务流程对齐：草稿、规划中、已排期、研发中、待验收、已上线运维、已关闭。
- 支持异常状态：已驳回、已挂起、已取消、变更中、退回修改。
- 每次状态变更需记录操作日志，包含前后状态、操作人、时间和说明。
- 支持按需求类型、来源角色、标签、优先级、状态、版本筛选。

### 6.3 版本隔离与比对

功能要求：

- 版本是需求、任务书、招投标文件、验收报告和预算的隔离边界。
- 切换版本时，需求列表、成果物列表、资金图表和指标卡需联动切换。
- 版本冻结后形成基线，冻结版本内的需求核心字段和成果物不可直接修改。
- 冻结后修改需提交变更申请，审批通过后才能落库。
- 支持跨版本比对：
  - 需求差异：新增、删除、修改高亮。
  - 预算差异：版本预算、需求分配预算、实际消耗对比。
  - 成果物差异：不同版本文件清单和版本号对比。

### 6.4 资金全链路追踪

资金追踪需实现四级穿透：

项目总预算 -> 年度预算 -> 版本预算 -> 需求预算/实际消耗。

功能要求：

- 提供资金拓扑图，可采用树状图、桑基图或分层预算流向图。
- 节点包含项目、年度、版本和需求。
- 颜色预警：
  - 超预算显示红色。
  - 预算剩余显示绿色。
  - 未分配或无预算显示灰色。
- 点击节点可查看下级明细。
- 提供计划预算、分配预算、实际消耗和执行率指标。
- 资金流水可关联到项目、年度、版本和具体需求。
- 修改预算金额需记录操作日志。

### 6.5 项目流程与成果物管理

功能要求：

- 以项目时间轴展示 6 个业务阶段。
- 每个阶段展示当前状态、关键节点、已挂载成果物和待办提醒。
- 成果物应按挂载对象过滤展示：
  - 项目下展示可研报告。
  - 年度计划下展示分年任务申报书。
  - 版本下展示任务书、招标文件、应标文件、验收报告、项目总结。
  - 需求下展示运维反馈、运营反馈和补充附件。
- 当版本下所有需求均进入待验收或已关闭状态时，提醒上传验收报告。
- 支持成果物目录导出。

### 6.6 报表、导出与备份

功能要求：

- 支持导出需求清单、资金明细、成果物目录。
- 支持按角色导出定制报表：
  - 客户版进展报告。
  - 销售版资金与进度报告。
  - 项目经理版交付与投入报告。
  - 研发版任务清单。
  - 运营版问题反馈清单。
- 支持本地数据备份和恢复。
- 备份需包含数据库和附件。

### 6.7 权限与审计

功能要求：

- 采用 RBAC 权限模型。
- 支持菜单级、字段级和数据行级权限。
- 客户不可查看内部成本、研发工时和敏感资金字段。
- 销售不可查看研发内部评估工时。
- 研发只看到自己负责或版本内可执行任务。
- 预算修改、状态变更、版本冻结、成果物上传、备份恢复均需写入操作日志。

## 7. UI/UX 要求

- 整体风格为企业级 SaaS：左侧导航，顶部上下文选择，右侧内容区。
- 首页应以指标卡、状态图、趋势图和任务列表组合展示，不应只有纯表格。
- 资金管理必须有可视化预算流向。
- 版本管理需有清晰的版本切换和隔离感。
- 表格需要支持筛选、排序和关键字段高亮。
- 关键操作要有确认、提示和失败反馈。
- 页面风格需统一，按钮、卡片、表格间距和颜色要一致。

## 8. MVP 优先级建议

### P0 必须完成

- 完整项目、年度、版本、需求、资金、成果物数据链路。
- 需求池和待规划池。
- 版本隔离视图。
- 资金四级穿透的基础可视化。
- 角色视图基础权限。
- 操作日志。

### P1 重要增强

- 跨版本需求和预算比对。
- 版本冻结后的变更申请。
- 阶段里程碑和成果物提醒。
- 角色定制报表。
- 标签管理。

### P2 后续优化

- 管理员拖拽配置看板。
- 更细粒度的数据行权限。
- 趋势图和周期统计。
- 多用户登录和密码策略。
- 审批流配置。
