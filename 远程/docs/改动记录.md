# 改动记录

## 2026-07-10

### 日志与正式运维增强

- 新增客户端 `runtime.log`、`error.log`、`audit.log` 三类分卷日志，审计采用 JSON Lines。
- 记录启动/停止、登录成功/失败、权限拒绝、MySQL 断连、SQL 摘要、事务失败和未捕获界面异常，不记录密码、SQL 参数或 OSS 凭据。
- 增加 `CRM_DATA_DIR`、`CRM_CONFIG_DIR` 和日志级别/大小/份数环境变量，支持只读安装目录。
- 应用关闭时主动关闭 MySQL 和日志句柄，系统设置可直接打开日志目录。
- 数据库操作日志新增对象类型、关键词过滤和审计 CSV 导出。
- 安装器同步创建 `data/logs` 和 `config`。
- 离线自测真实验证运行日志、错误分流和审计 JSON。
- 新增 `healthcheck.py`，用于目标环境检查 MySQL、关键表、目录、磁盘以及服务器附件/OSS 写删能力。
- 成果物列表支持打开服务器共享附件或下载 OSS 附件，并记录操作审计；服务器路径越界会被拒绝。
- 新增仓库根目录 `日志与运维说明.md`，明确多客户端集中采集、告警、留存和 MySQL 服务端日志边界。
- 修复 `CRM_LOG_LEVEL=CRITICAL` 导致 `error.log` 漏记普通错误；操作日志新增 `event_id/result`，可与 JSON 文件审计对账。
- 审计事件 ID 改为 UUID4 并增加唯一索引；旧记录标记 `legacy`，MySQL 结构升级使用 `GET_LOCK` 防止多客户端并发 DDL。
- 登录失败、权限拒绝和健康检查失败同时记录中央 MySQL 操作日志与本地审计，中央写入失败时保留客户端兜底审计。
- 远程自测默认只执行离线检查，只有 `CRM_MYSQL_INTEGRATION_SELFTEST=1` 才连接测试库，避免误写生产。

### 生产一致性与附件安全复核

- 需求新增、编辑、版本分配和软删除统一锁定版本行，避免与版本冻结并发时产生基线遗漏。
- 变更申请头、payload 和日志改为同一事务，缺少 payload 的申请禁止审批通过。
- 成果物元数据和操作日志改为同一 MySQL 事务；数据库失败时补偿删除已上传对象，界面刷新失败不再误删有效附件。
- 服务器附件改存相对对象键，允许不同客户端使用不同挂载点；附件根拒绝磁盘根、共享根、用户主目录和受保护目录祖先。
- OSS 强制 HTTPS，下载/删除限制到当前业务前缀，环境凭据轮换后重建连接。
- 增加 MySQL 读写超时；健康检查验证 MySQL 8、`utf8mb4`、严格 SQL mode、TLS cipher 和附件盘空间。
- `mysql-connector-python` 最低版本提高到 9.2.0，与读写超时参数保持兼容；TLS/严格模式环境开关拼写错误时直接失败。
- 需求跨版本分配增加项目一致性校验；成果物事务内重新锁定并校验挂载对象。
- 所有显式事务统一恢复游标和 `autocommit`，避免游标创建/关闭异常污染后续连接状态。
- `skip_login` 仅允许显式集成测试；管理员首次密码更新增加并发条件。
- ZIP 备份跳过符号链接，恢复增加解压总量、临时盘空间和异常压缩比检查。
- 修复安装器 spec 的机器绝对路径，远程主程序打包显式包含 `oss2`。
- MySQL 集成自测新增需求新增/编辑/分配/冻结/变更审批/软删除和成果物挂载事务测试，仅在测试库开关开启时执行。

### 仓库同步

- 按要求以远程 `origin/main` 覆盖本地仓库，基线提交为 `07f1453 Split local and MySQL remote versions`。

### MySQL 正式环境能力

- MySQL 连接增加超时、连接存活检查、自动重连和自动提交，减少长连接陈旧快照。
- 通用写操作执行后立即关闭 MySQL 游标，仅返回行数和新增 ID，避免长时间运行积累游标资源。
- 支持 `ssl_ca`，配置后强制校验证书和服务器身份。
- 生产默认 `create_database=false`、`seed_demo_data=false`，演示数据仅显式开启时初始化。
- 数据库密码支持从 `CRM_DB_PASSWORD` 环境变量读取，配置模板不再要求明文密码。
- 预算登记增加显式事务、`SELECT ... FOR UPDATE` 行锁、版本预算汇总锁和异常回滚。
- 版本冻结、基线、需求快照、冻结状态和日志改为单一事务。
- 变更审批、需求修改/软删除、状态历史、审批状态和日志改为单一事务。

### 登录、账号和权限

- 增加远程版登录窗口和首次管理员密码初始化，支持 `CRM_INITIAL_ADMIN_PASSWORD`。
- 密码使用随机盐 PBKDF2-HMAC-SHA256 哈希保存。
- 增加用户创建、启用/停用、本人修改密码和管理员重置密码；禁止停用当前登录账号。
- 增加客户项目授权表和管理员授权/撤销操作；客户项目上下文、项目列表、首页项目数和搜索按白名单过滤。
- 角色由登录用户决定，移除可伪造的角色选择器。
- 客户、研发人员、运营人员隐藏资金页面和资金字段；导出、系统设置、用户管理和备份恢复按角色限制。

### 需求、版本和资金

- 增加受控状态机、状态历史、流转说明必填和并发条件更新；状态、历史和日志在同一事务提交。
- 增加预算有限数值、非负数、层级上限、需求关联、版本超分配和实际超支校验。
- 冻结版本仅允许继续登记实际消耗，预算分配和调整在数据库事务层被拒绝。
- 增加年度合理区间、`YYYY-MM-DD` 有效日期和版本起止日期顺序校验。
- 普通需求编辑不再直接修改已分配预算和实际消耗。
- 冻结版本需求的修改/删除转入审批，审批通过后真正应用。
- 跨版本比对按标准化需求名称匹配；冻结版本阻止需求直接移出。

### 附件双存储

- 新增 `attachment_storage=server|oss` 配置。
- 服务器模式支持 Linux 本机目录或客户端可访问的 SMB/NFS 共享目录。
- OSS 模式支持 endpoint、Bucket、前缀和 STS；AccessKey 只从环境变量读取。
- 新增 `oss2` 依赖；OSS 路径按 `oss://bucket/key` 保存。
- 上传前校验挂载对象，数据库失败时清理服务器文件或 OSS 对象；删除服务器文件时限制在附件根目录内。
- 远程 ZIP 不再包含数据库密码或 MySQL 数据；服务器附件与 MySQL 分别备份，OSS 使用 Bucket 版本控制。
- 服务器附件恢复改为目标目录同盘暂存和失败自动回滚，并拒绝把附件目录配置为磁盘根、共享根或应用根目录。

### 页面、测试和文档

- 增加长弹窗滚动、预算状态提示、冻结提示、金额排序和 CSV 公式注入防护。
- 修复成果物阶段统计、文件名碰撞和附件 ZIP 路径穿越问题。
- `selftest.py` 增加密码哈希、CSV、状态机和权限离线检查；1.5.0 起默认离线，集成测试改用显式测试库开关。
