# 版本变更记录

## v1.5.0-mysql - 2026-07-10

### 正式部署加固

- 中央操作日志新增 `event_id/result`，登录失败、权限拒绝和健康检查失败可与客户端 JSON 审计对账。
- 事件 ID 改为 UUID4 唯一索引；旧日志标记 `legacy`，结构迁移增加跨客户端命名锁。
- 需求新增、编辑、版本分配、删除、变更申请和成果物元数据写入改为事务化，需求写入与冻结共用版本行锁。
- 服务器附件保存相对对象键；OSS 强制 HTTPS、业务前缀限制和凭据轮换重建连接。
- 健康检查增加 MySQL 8、字符集、严格模式、TLS cipher、读写超时和附件盘检查。
- Connector 最低版本调整为 9.2.0；布尔安全开关写错即失败，事务结束统一恢复连接状态。
- 分配需求增加项目不变量校验，成果物在元数据事务内重新校验并锁定挂载对象。
- 自测默认离线，MySQL 集成测试必须显式设置测试开关；登录跳过只允许该测试模式。
- ZIP 恢复增加符号链接、解压容量和压缩比限制；修复可复现打包路径并包含 OSS 依赖。

### 升级说明

- 旧库首次启动会为 `operation_logs` 增加 `event_id/result`，需要一次性 `ALTER` 权限；升级验证后可撤销。
- 目标 Linux MySQL、TLS、共享目录/OSS、并发和恢复仍须按验收文档完成真实环境测试。

## v1.4.1-mysql - 2026-07-10

### 正式运维

- 新增客户端运行、错误、审计三类文件日志及大小轮转。
- 登录成功/失败、权限拒绝、MySQL 断连、数据库失败和关键事务均有分类日志。
- 审计日志采用 JSON Lines，包含客户端主机、进程、版本、操作者和结果，不记录密码、SQL 参数或 OSS 密钥。
- 增加 `CRM_DATA_DIR` 和 `CRM_CONFIG_DIR`，支持安装目录只读的正式部署。
- 系统设置显示日志位置并可打开目录；应用退出时主动关闭 MySQL 和日志句柄。
- 操作日志支持按类型/关键词过滤和受权限控制的 CSV 导出。
- 新增集中日志采集、告警、MySQL 服务端日志和留存说明。

### 测试

- 远程离线自测验证日志分流、错误输出和审计 JSON。
- 新增 MySQL 与附件后端真实部署健康检查脚本。
- 新增服务器附件打开、OSS 附件下载和附件访问审计。
- 目标 Linux MySQL、TLS、OSS/共享目录和多客户端日志集中采集仍按上线清单验收。

## v1.4.0-mysql - 2026-07-10

### 新增

- 新增账号登录、首次管理员初始化、本人改密、管理员重置密码和账号停用。
- 新增客户项目白名单和管理员授权/撤销操作，避免客户枚举未授权项目。
- 新增服务器共享目录和阿里云 OSS 两种附件存储后端。
- 新增版本基线、需求状态历史、变更载荷和完整审批应用。
- 新增 Linux MySQL、TLS、最小权限、附件和上线验收文档。

### 并发与安全

- 预算登记使用 MySQL 事务和行锁，防止多客户端丢失更新。
- 版本冻结和变更审批改为原子事务。
- 需求状态流转、历史和日志改为原子事务。
- 冻结版本禁止继续分配或调整预算，只允许登记实际消耗。
- 支持 MySQL 密码、OSS AccessKey 和首次管理员密码环境变量。
- 支持 MySQL CA 证书和服务器身份校验。
- 生产默认关闭自动建库和演示数据。
- 客户、研发、运营隐藏资金信息，导出、系统设置、用户管理和备份恢复按角色收紧。

### 优化与修复

- 增加受控状态机、预算层级校验、非有限金额拒绝和 CSV 公式注入防护。
- 增加年度、有效日期格式和版本起止顺序校验。
- 修复冻结审批未真正应用需求修改/删除、成果物层级统计和附件异常遗留。
- 连接增加超时、存活检查和自动重连。
- 通用写操作自动关闭 MySQL 游标，降低长时间运行的连接资源占用。
- 服务器附件删除限制在配置根目录，OSS 备份边界明确为 Bucket 运维策略。
- 服务器附件恢复使用同盘暂存和失败自动回滚，危险根目录配置会被拒绝。

### 测试

- 离线密码哈希、状态机、权限和 CSV 安全检查通过。
- MySQL/OSS 在线集成与双客户端并发测试需在目标 Linux 环境按验收文档执行。

## v1.1.0 - 2026-06-08

### 新增

- 新增 `ConsultingRequirementManagerSetup.exe` 安装器。
- 双击安装器后可选择安装路径。

- 安装完成后可选择立即启动主程序。
- 源码新增 `installer.py` 和 `ConsultingRequirementManagerSetup.spec`。

### 调整

- `exe` 目录调整为只保留安装器，便于用户直接双击安装。
- `source` 目录保留源码、README、打包配置和 Git 仓库。
- README 增加安装器打包说明。

### 说明

- 主程序仍为 `ConsultingRequirementManager.exe`，由安装器复制到用户选择的安装目录。
- 运行数据默认保存在安装目录下的 `data` 文件夹。
