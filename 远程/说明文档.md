# 咨询项目全流程需求管理系统 - MySQL 远程版

这是面向多人共享的桌面客户端版本。Linux 服务器运行 MySQL 8.x；Windows/Linux 桌面客户端通过内网或 VPN 连接。附件可统一选择服务器共享目录或阿里云 OSS。

## 当前能力

- MySQL 存储项目、年度、版本、需求、资金、成果物、用户、日志、基线和变更申请。
- 用户名密码登录，密码保存为随机盐 PBKDF2-HMAC-SHA256 哈希。
- 首次管理员初始化、本人修改密码、管理员创建/停用用户和重置密码。
- 客户账号默认无项目权限，管理员可按当前项目授权或撤销，客户列表和搜索只显示已授权项目。
- 角色工作台、敏感资金字段隐藏、受控状态机、全局搜索和角色受限导出。
- 待规划需求池、版本隔离、冻结基线、变更审批和跨版本比对。
- 资金四级穿透、预算预警和 MySQL 行锁事务，避免并发丢失更新。
- 六阶段里程碑、成果物挂载和验收材料提醒。
- 附件支持 `server` 共享目录或阿里云 `oss`，OSS 凭据只从环境变量读取。

## 快速启动

```powershell
python -m pip install -r requirements.txt
Copy-Item config\mysql_config.example.json config\mysql_config.json
$env:CRM_DB_PASSWORD = "数据库强密码"
$env:CRM_INITIAL_ADMIN_PASSWORD = "首次管理员强密码"
python app.py
```

生产配置保持：

```json
{
  "password": "",
  "password_env": "CRM_DB_PASSWORD",
  "create_database": false,
  "seed_demo_data": false,
  "ssl_ca": "C:/secure/mysql-ca.pem",
  "attachment_storage": "oss"
}
```

真实配置 `config/mysql_config.json` 已被 Git 忽略。不要把数据库密码或 OSS AccessKey 写入仓库。

## 检查

无需 MySQL 的离线检查：

```powershell
$env:CRM_OFFLINE_SELFTEST = "1"
python selftest.py
Remove-Item Env:CRM_OFFLINE_SELFTEST
```

连接测试 MySQL 后执行完整页面检查：

```powershell
python selftest.py
```
