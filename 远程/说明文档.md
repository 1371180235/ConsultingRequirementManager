# 咨询项目全流程需求管理系统 - MySQL 8.x LTS 远程版

当前客户端应用版本为 v1.6.3；它不是 MySQL 数据库版本。远程版要求 MySQL 8.x，推荐使用 MySQL 8.4 LTS。

这是面向多人共享的桌面客户端版本。Linux 服务器运行 MySQL 8.x；Windows/Linux 桌面客户端通过内网或 VPN 连接。附件可统一选择服务器共享目录或阿里云 OSS。

## 当前能力

- MySQL 存储项目、年度、版本、需求、资金、成果物、用户、日志、基线和变更申请。
- 用户名密码登录，密码保存为随机盐 PBKDF2-HMAC-SHA256 哈希。
- 首次管理员初始化、本人修改密码、管理员预建账号、修改角色、启停账号、重置密码和强制下线；不开放自助注册。
- 同一账号仅保留最后一次登录会话，旧客户端最长 10 秒内自动退出；旧客户端关闭不会影响新会话。
- 客户账号默认无项目权限，管理员可按当前项目授权或撤销，客户列表和搜索只显示已授权项目。
- 角色工作台、敏感资金字段隐藏、受控状态机、全局搜索和角色受限导出。
- 待规划需求池、版本隔离、冻结基线、变更审批和跨版本比对。
- 支持版本发布自动基线、稳定业务需求标识、标签字典和多选标签。
- 支持研发任务领取、预估/实际工时、工时明细和角色专用项目报告。
- 支持管理员拖动配置各角色看板，首页提供需求推进趋势图。
- 支持云岚蓝、松石青、靛夜紫、石墨灰四套主题顶部一键切换，并按登录账号保留选择。
- 资金四级穿透、预算预警，以及覆盖预算、冻结、需求写入和变更申请的 MySQL 行锁事务。
- 六阶段里程碑、成果物挂载和验收材料提醒。
- 附件支持 `server` 共享目录或阿里云 `oss`；服务器文件保存相对对象键，OSS 强制 HTTPS 和业务前缀校验，凭据只从环境变量读取。
- 支持客户端运行、错误、审计三类轮转日志，登录失败、权限拒绝、数据库断连和未捕获异常均可追踪。
- 支持 `CRM_DATA_DIR` 和 `CRM_CONFIG_DIR`，适配只读安装目录。

## 快速启动

```powershell
python -m pip install -r requirements.txt
Copy-Item config\mysql_config.example.json config\mysql_config.json
$env:CRM_DB_PASSWORD = "数据库强密码"
$env:CRM_INITIAL_ADMIN_PASSWORD = "首次管理员强密码"
$env:CRM_DATA_DIR = "D:\ConsultingRequirementManagerData"
$env:CRM_CONFIG_DIR = "D:\ConsultingRequirementManagerConfig"
$env:CRM_REQUIRE_TLS = "1"
python app.py
```

生产配置保持：

```json
{
  "password": "",
  "password_env": "CRM_DB_PASSWORD",
  "create_database": false,
  "seed_demo_data": false,
  "read_timeout": 30,
  "write_timeout": 30,
  "ssl_ca": "C:/secure/mysql-ca.pem",
  "attachment_storage": "oss"
}
```

真实配置 `config/mysql_config.json` 已被 Git 忽略。不要把数据库密码或 OSS AccessKey 写入仓库。
`server` 附件模式必须填写所有客户端可访问的 SMB/NFS/UNC 共享目录；留空会拒绝启动。
仅测试或演示环境可临时启用 `seed_demo_data=true`，系统会生成两个带差异需求的版本以便验证跨版本比对；生产环境必须保持 `false`。

## 检查

无需 MySQL 的离线检查（默认行为，不会连接正式库）：

```powershell
python selftest.py
```

连接测试 MySQL 后显式执行完整页面检查，禁止指向生产库：

```powershell
$env:CRM_MYSQL_INTEGRATION_SELFTEST = "1"
python selftest.py
Remove-Item Env:CRM_MYSQL_INTEGRATION_SELFTEST
python healthcheck.py
```
