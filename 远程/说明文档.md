# 咨询项目全流程需求管理系统 - MySQL 远程版

这是面向多人/多电脑共享数据场景的 MySQL 版本。界面和业务功能沿用本地版，但业务数据存储在 MySQL 中，适合部署到服务器或局域网数据库。

## 当前能力

- 使用 MySQL 存储项目、年度计划、落地版本、需求任务、资金流水、成果物、用户、日志和变更申请。
- 首次连接后自动创建业务表，并初始化演示数据。
- 支持角色视角、需求待规划池、版本隔离、资金四级穿透、成果物管理、流程里程碑、搜索和 CSV 导出。
- 附件、导出和 ZIP 备份仍保存在应用所在机器的 `data/` 目录。

## 快速启动

```powershell
pip install -r requirements.txt
Copy-Item config\mysql_config.example.json config\mysql_config.json
notepad config\mysql_config.json
python app.py
```

详细 MySQL 创建用户、授权、远程访问和备份说明见：

[docs/mysql部署说明.md](docs/mysql部署说明.md)

## 配置文件

运行前需要配置：

```json
{
  "host": "127.0.0.1",
  "port": 3306,
  "user": "crm_user",
  "password": "你的数据库密码",
  "database": "consulting_requirement_manager",
  "create_database": true
}
```

真实配置文件 `config/mysql_config.json` 已被 `.gitignore` 忽略，不会提交数据库密码。

## 可运行性检查

静态检查：

```powershell
python -m py_compile app.py selftest.py installer.py
```

连接 MySQL 后做页面冒烟测试：

```powershell
python selftest.py
```
